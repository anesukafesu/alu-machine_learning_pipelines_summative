Gather data

A machine learning model is only as good as the data it learns from. As the old adage goes, garbage in, garbage out. High-quality data is the foundation of a robust model—poor or incomplete data can lead to inaccurate predictions and unreliable results.

The features

The dataset has the following features:

  • person_home_ownership: Represents the person's home ownership status. A person can either own, rent, or mortgage their home.
  • loan_int_rate: The interest rate on the loan.
  • loan_percent_income: The loan as a percentage of income. This is useful for determining the debt burden of any loan.
  • cb_person_default_on_file: Whether or not the person has ever defaulted on a loan before.

The target

The model predicts whether a person is going to default on their loan or not. In the current dataset, the number of non-defaults far outweighs the number of defaults. Therefore, during training, oversampling is used to balance the classes and improve prediction performance.

New data

Additionally, gathering data is not a one-time process. It’s important to iteratively add new data over time, as this helps improve your model’s performance and adaptability to changes in the problem domain. You can add data in the csv format. Each row should be recorded in its own line. Additionally, each row should have the following values separated by commas: person_home_ownership, loan_int_rate, loan_status, loan_percent_income, cb_person_default_on_file.

Model Training

In this pipeline, we’ll be using a Logistic Regression model to perform predictions. Logistic regression is a widely used algorithm for binary classification problems, making it an excellent choice for this dataset. Preconfigured Model The logistic regression model has been preconfigured for your convenience. All the necessary parameters have been set up, ensuring it is ready to learn from your data. You can focus on understanding the process without worrying about the underlying configuration details.

Model Evaluation

After training a model, the next critical step is evaluation. This process helps us understand how well the model performs on new, unseen data by testing its ability to make accurate predictions. While there are many metrics, f1 score offers a comprehensive view, therefore that is what we use to rank models.
